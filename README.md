# Tower Defence

`Tower Defence` это tower defense проект на `Unity`, в котором игровая логика, конфигурация данных, создание сущностей и UI разделены по отдельным подсистемам.

Проект использует `Zenject`, `UniTask`, `DOTween`, `NavMesh` и конфиги на `ScriptableObject`.

## Обзор

Игровой цикл организован так:

1. Bootstrap-сцена загружает глобальные сервисы, прогресс и конфиги.
2. Сцена `Menu` служит входной точкой в игру.
3. Сцена `Level` собирает runtime-зависимости через installers.
4. Перед каждой волной запускается countdown UI.
5. Игрок размещает башни в допустимых точках.
6. Башни создают оружие и атакуют врагов.
7. Враги спавнятся волнами, двигаются к замку и переключают поведение через state machine.
8. После завершения уровня игрок получает мета-валюту, которая сохраняется в прогрессе.

## Архитектура

Проект разделен на несколько основных зон:

- `Infrastructure` содержит bootstrap, загрузку сцен, константы, pooling и базовые элементы state machine
- `Logic` содержит игровой цикл, волны, здоровье, монеты и presenter-логику
- `Services` содержит работу с ассетами и сохранениями
- `ConfigRepositories` предоставляет доступ к игровым конфигам
- `Enemy`, `Tower`, `Weapon` содержат доменные игровые сущности
- `UI` содержит панели, экраны и view-компоненты

Такое разделение отделяет игровую логику от сценовых зависимостей и prefab-части.

## Используемые решения

### Dependency Injection

Композиция зависимостей строится через `Zenject`:

- `ProjectContextInstaller` связывает глобальные сервисы и prefab databases
- `BootstrapSceneInstaller` запускает стартовую инициализацию
- `LevelSceneInstaller` собирает зависимости уровня
- для игровых объектов используются отдельные installers

### Конфиги и данные

Параметры игры вынесены в `ScriptableObject`:

- `GameConfig`
- `TowerConfig`
- `EnemyConfig`
- `WeaponConfig`

Загрузка идет через `AssetProvider` и репозитории:

- `GameRepository`
- `TowerConfigsRepository`
- `EnemyConfigsRepository`
- `WeaponConfigsRepository`

### Создание сущностей

Создание runtime-объектов централизовано:

- `EnemyFactory`
- `TowerFactory`
- `WeaponFactory`
- `UIFactory`

### Pooling

Для врагов используется Zenject memory pool:

- `EnemyPool`

### State Machine

В проекте есть собственная реализация state machine:

- `IState`
- `ITransition`
- `StateMachine`
- состояния врагов: движение, idle, атака, смерть
- состояния замка: обычное состояние и разрушение

### Event-driven взаимодействие

Часть систем взаимодействует через события:

- изменение здоровья
- смерть сущности
- обновление монет
- запуск countdown волны
- завершение уровня
- выбор точки размещения башни

### Асинхронная логика

`UniTask` используется для:

- загрузки сцен
- таймера между волнами
- задержек спавна врагов
- отмены активных wave-flow операций через `CancellationToken`

## Основные подсистемы

### Bootstrap

`GameBoostrapper`:

- загружает прогресс игрока
- загружает игровой конфиг
- загружает конфиги башен
- загружает конфиги врагов
- загружает конфиги оружия
- открывает сцену `Menu`

### Меню

`MenuCanvas` загружает сцену `Level`.

### Уровень

`GameManager`:

- создает HUD
- создает панель выбора башен
- подписывается на события размещения башен
- запускает countdown до волны
- создает замок
- обрабатывает победу и поражение

### Волны

`WaveManager`:

- хранит индекс текущей волны
- запускает countdown
- спавнит врагов по данным из `GameConfig`
- считает убитых врагов
- рассчитывает награду
- завершает уровень после последней волны

### Размещение башен

`TowerPlacement`:

- обрабатывает ввод мыши и тача
- игнорирует клики по UI
- проверяет занятые области через raycast
- передает позицию для создания башни

### Боевая логика

- башни создаются через `TowerFactory`
- оружие подключается через `WeaponFactory`
- враги создаются через `EnemyFactory`
- здоровье хранится в `HealthModel`
- монеты хранятся в `CoinCounterModel`

## Расширение проекта

Текущая структура позволяет отдельно расширять:

- типы врагов
- типы башен
- типы оружия
- конфиги волн
- UI-панели
- поля прогрессии в `PlayerProgress`
- набор сцен

Для добавления нового контента используются конфиги, prefab databases, фабрики и scene/object installers.

## Запуск

1. Открыть проект в `Unity 6.0 / 6000.4.0f1`.
2. Дождаться восстановления пакетов.
3. Открыть bootstrap-сцену проекта.
4. Запустить `Play Mode`.

Ожидаемый flow:

- инициализация сервисов и конфигов
- загрузка меню
- переход на сцену уровня
- запуск countdown до первой волны

## Примечания

- Конфиги загружаются через `Resources`.
- Прогресс сохраняется через `PlayerPrefs`.
- В репозитории есть незакоммиченные ассеты, prefab и package-related файлы. README описывает текущее состояние исходников в рабочей директории.
